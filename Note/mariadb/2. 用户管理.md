
# 创建用户

```sql
CREATE USER 'shins0u'@'localhost' IDENTIFIED BY 'password';
CREATE USER 'shins0u'@'%' IDENTIFIED BY 'password';
```

第二行使用 % 来匹配所有主机名, 使网络中其他主机可使用此用户名访问数据库

# 删除用户

```sql
DROP USER shins0u@'%';
```

# 修改用户密码

```sql
ALTER USER 'root'@'localhost' IDENTIFIED BY '114514';
-- 双引号: 有特殊符号, 例如这里有 .
ALTER USER "root_"@'127.0.0.1' IDENTIFIED BY '114514';
-- 不加引号: 仅合法标识符: 无特殊符号,空格等
alter user 'root'@localhost IDENTIFIED BY '114514';
--
SET PASSWORD for root@localhost = PASSWORK('mariadb')
```

| 对象          | 是否区分大小写  | 例子                              |
|---------------|-----------------|-----------------------------------|
| SQL 关键字    |   不区分       | select 等同于 SELECT              |
| 字段名 (列名) |   不区分       | user_id 等同于 USER_ID            |
| 表名 / 库名   |   区分 (Linux) | User 不等同于 user                |
| 别名 (Alias)  |   不区分       | AS u 等同于 AS U                  |
| 字符串内容    |   看字符集配置 | 默认不区分(_ci), 二进制区分(_bin) |

ci: Case Insensitive

- Q: 为什么数据库名,表名在 linux 上大小写敏感?
- A: 底层使用文件系统存储数据, 而linux的文件系统一般大小写敏感, win/mac 不会有这种情况

# 重载权限配置

```sql
FLUSH PRIVILEGES;
```

这些情况不需要手动重载:

- CREATE USER
- DROP USER
- ALTER USER
- GRANT
- REVOKE

这是合法的修改方式
