# 数据库的三级模式

外模式: 也称子模式,用户模式, 是
数据库用户(程序员,程序用户)能够看见和使用的局部数据的逻辑结构和特征的描述, 是
数据库用户的数据视图, 是
与某一应用有关的数据的逻辑表示

外模式是模式的子集, **一个数据库可有多个外模式**

模式: 也称逻辑模式, 是
数据库中全体数据的逻辑结构和特征的描述, 是
所有数据的公共视图, 是
数据库系统模式结构的中间层, **一个数据库只有一个模式**

内模式: 也称存储模式, 是
数据库结构与存储方式的描述, 是
数据库在数据库内部的表示方法, 例如记录的存储方式是堆存储还是升降序存储, **一个数据库只有一个模式**

外模式描述数据, 模式描述数据的逻辑结构和特征, 内模式描述数据在计算机上如何存储(c盘,d盘)

# 数据库的两级映像

外模式/模式映像: 定义了外模式和模式之间的对应关系, 当模式改变时(新增关系, 属性, 改变属性类型等), 外模式也改变

如果不希望外模式改变, 需要数据库管理员对各个外模式/模式映像作相应改变, 可以使
外模式保持不变, 保证了数据与程序的 **逻辑独立性**

模式/内模式映像: 定义了模式和内模式之间的对应关系, 当存储结构改变时, 需要对模式/内模式映像作相应改变, 可以
使模式保持不变, 从而应用程序也不必改变, 保证了数据的 **物理独立性**
